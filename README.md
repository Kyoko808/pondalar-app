# Pondalar アプリの修正を Git に反映する手順

Pondalar アプリのコードを更新したら、以下の手順で Git リポジトリに反映させてください。

## 1. 変更内容を確認する
```bash
git status -sb
```
`M app.py` のように、修正したファイルが表示されていることを確認します。差分も確認したい場合は `git diff` を使います。

## 2. 変更をステージングする
```bash
git add app.py README.md
```
修正したファイルを `git add` でステージングします。複数のファイルをまとめて追加する場合は `git add .` でも構いません。

## 3. コミットメッセージを付けてコミットする
```bash
git commit -m "Improve chat handling and search tabs"
```
以前の修正内容に沿ったメッセージを付けると、履歴を追いやすくなります。必要に応じてメッセージを調整してください。

## 4. リモートリポジトリへプッシュする
```bash
git push origin work
```
作業中のブランチ（例：`work`）をリモートへプッシュします。別のブランチ名を使っている場合は適宜読み替えてください。

## 5. Pull Request を作成する
GitHub などのホスティングサービス上で Pull Request を作成し、変更内容とテスト結果を記載します。レビューコメントが付いた場合は、同様の手順で修正・再コミット・再プッシュすれば更新できます。

---
これで、今回の修正を含む変更が Git に反映され、共有できるようになります。
